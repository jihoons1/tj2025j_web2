[ ORM ] 0:object , R : Relational ,  M : Mapping : 객체관계매핑 *기술*
    1. 정의 : 객체지향 프로그래밍에서 사용되는 객체 와 DB(관계형데이터베이스) 매핑(연결)
    2. 목적 : 편리성 제공
    3. 사용처 : MYBATIS, JPA 등등

[ Hibernate ] :
    1. 정의 : 자바에서 ORM 구현한 라이브러리
    2. 목적 : SQL 없이 순수 자바만으로 데이터베이스 처리한다.
    3. 기능 : 1. CRUD 2. 타입변환 3. 부가기능(검색/정열/페이징처리) 등등


[ JPA ] J:java , P : persistence , A : api
    자바 영속성/지속성/고집 API
    1. 정의 : 자바에서 ORM 사용할때 표준 API , 자바에서 다양한 라이브러리 제공(+Hibernate)
    2. 설치 : implementation 'org.springframework.boot:spring-boot-starter-data-jpa'

[ Entity ]
    1. 데이터베이스 테이블과 매핑된 클래스
    Entity클래스 =DB테이블 , Entity객체 = 레코드
        - 클래스 = 객체의 설계도
        -class Member{
            int id;
            String name;
        }
        - 객체 = 클래스의 인스턴스
        - new Member( 1 , "유재석" )
    2. 사용법
        1) 선언 : 클래스 위에 @Entity 주입한다.
        + 엔티티 1개당 @Id 필수다 . ( primary key)
        2) Entity 매핑 설정 , application.properties

[ 리포지토리 ] JPA Repository
    1. 정의 : 기본적인 CRUD 작업을 제공하는 인퍼에스 , +동적쿼리
    2. 사용법 : extends JapRepository< T , ID > 상속 받는다.
